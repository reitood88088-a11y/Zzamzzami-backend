**기술 스택** 

웹에서 제스처 UI(릴스/스토리 형식)를 구현하고 외부 AI 및 OCR API를 효율적으로 처리할 수 있는 Next.js + FastAPI + PostgreSQL 조합을 제안한다.

---

#### 1. 프론트엔드: Next.js (React) + TypeScript

웹 브라우저에서 '앱과 같은 경험' 가능

* **OpenAPI 명세 기반 API 동기화**: 백엔드(FastAPI)에서 자동 생성되는 OpenAPI(Swagger) 명세를 활용하여 프론트엔드 타입과 API 클라이언트를 손쉽게 동기화할 수 있다.
* **검증된 터치/제스처 라이브러리 생태계:** React 생태계의 터치 라이브러리(예: `Swiper.js`, `Framer Motion`)를 활용하면 한 손 조작 UI를 구현할 수 있다.
* **웹 카메라 API 안정성:** 모바일 앱의 카메라 기능 대신 웹 표준인 `HTML5 Camera API (getUserMedia)`를 사용해야 한다. Next.js(React) 환경에서는 이와 관련한 오픈소스 래퍼 라이브러리가 있다.

#### 2. 백엔드: FastAPI (Python)

Vercel 서버리스 배포 환경 최적화 및 AI/데이터 처리 파이프라인 연동에 탁월한 선택이다.

- **서버리스 최적화 및 빠른 콜드 스타트:** FastAPI는 서버리스 환경에서 콜드 스타트가 빠르다. 
- **AI/OCR 생태계 통합의 용이성:** 파이썬 환경의 강력한 데이터 처리 라이브러리
- **대기 시간이 긴 API의 안정적 처리:** Pydantic 기반의 강력한 데이터 검증과 비동기(asyncio) 처리한다.

#### 3. 데이터베이스: PostgreSQL (+ SQLAlchemy / SQLModel)

- **안정적이고 가벼운 ORM 적용:** 가벼운 SQLAlchemy(또는 SQLModel)를 사용해 서버리스 환경의 용량 제한 및 커넥션 이슈를 최소화한다.
- **세션 유지 및 유연한 퀴즈 저장:** 사용자가 웹 브라우저를 새로고침하거나 잠시 이탈해도 마지막 학습 위치를 기억하는 '학습 이어하기' 기능을 관계형 데이터로 보장합니다. 동시에 AI 변형 퀴즈 데이터는 JSONB 타입으로 저장하여 스키마 변경 에러 없이 유연하게 대응합니다.

###  웹 구현 시 에러를 줄이는 핵심 팁

- **카메라 촬영의 대안 마련:** 웹 환경은 기기나 브라우저에 따라 카메라 권한 거부 에러가 잦다. 따라서 카메라 직접 촬영뿐만 아니라 **'갤러리에서 이미지 파일 업로드(Input)' 기능**을 무조건 교차 지원해야 한다.
- **로컬 스토리지 활용:** 네트워크가 잠시 끊기거나 브라우저가 꺼져도 사용자의 세션(학습 데이터)이 날아가지 않도록, 프론트엔드단에서 `localStorage`나 `IndexedDB`에 임시 저장하는 로직을 두는 것이 좋다.
